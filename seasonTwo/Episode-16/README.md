- Now we will store the data into `redux store`. The Redux store is a single, centralized object that holds the entire global state of a JavaScript application. It acts as the "single source of truth" for the application's data, providing a predictable way to manage how that state changes over time.
  - [Redux](https://redux.js.org/) can be used for any application that places a high priority on the ability to store states. It should be clear that Redux can be used for the client side (frontend) with user interfaces. However, since Redux is just JavaScript, it can also be used on the server side (backend).
  - _Redux Toolkit is our official recommended approach for writing Redux logic. It wraps around the Redux core, and contains packages and functions that we think are essential for building a Redux app. Redux Toolkit builds in our suggested best practices, simplifies most Redux tasks, prevents common mistakes, and makes it easier to write Redux applications._
  - [Redux Toolkit](https://redux-toolkit.js.org/) - Add the Redux Toolkit and React-Redux packages to your project - [Usage Instructions](https://redux-toolkit.js.org/tutorials/quick-start) - `pnpm add @reduxjs/toolkit react-redux`

```md
### Key Responsibilities of the Redux Store

The store is the core of Redux and comes with a few essential methods for interacting with the application state:

- `Holds the current application state`: The global state is stored internally within the store as a single object tree.

- `getState()`: Allows components to access the current state of the application.

- `dispatch(action)`: This is the only way to trigger a state change. An action is a plain JavaScript object describing what happened in the application, and dispatching it signals an intent to update the state.

- `subscribe(listener)`: Registers a callback function that will be executed any time an action is dispatched and the state might have changed. This allows the UI to update with the new data.
```

- Now this store can consists of `Slices` - A "slice" in Redux is a term for the portion of the Redux state and its corresponding logic (reducers and actions) that relates to a single feature of your application. The name comes from the idea of splitting the single, large Redux state object into multiple, manageable "slices" of state.

```md
### Key Concepts of a Redux Slice

- `Bundles Logic`: A slice combines the initial state, the reducer logic, and the action creators for a specific feature (e.g., "users", "posts", "counter") into a single file or module.
- `createSlice` Function: This utility function simplifies the traditional Redux setup by automatically generating action type strings and action creator functions based on the names of the reducer functions you provide.
- `Immutable Updates Made Easy`: Internally, createSlice uses a library called Immer which allows you to write state updates as if you were directly mutating the state (e.g., state.value = 1), but it actually translates these into safe, immutable updates behind the scenes.
- `Modularity`: Slices promote modularity and separation of concerns by keeping all the related state management logic for a feature in one place, making the code easier to organize, read, and maintain.
- `Integration`: The reducer function generated by a slice is then exported and combined with other slice reducers to form the single "root reducer" that is passed to the Redux store.
```
